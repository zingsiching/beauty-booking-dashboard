# 预约会员看板数据库接口 SOP

## 一、交付物

本阶段交付两样东西：

1. 完整看板页面：`index.html`
2. Google Sheets 数据库接口 SOP：本文档

配套文件：

- `google-apps-script.gs`：复制到 Google Apps Script 的接口代码。
- `google-sheets-storage-plan.md`：给客户看的储存逻辑说明。
- `google-sheets-setup-guide.md`：详细设置步骤。

## 二、数据库结构

Google Sheets 文件建议命名：

`门店预约与会员管理数据库`

包含三个工作表：

- `预约记录`
- `会员资料`
- `系统设置`

## 三、预约记录表用途

用于储存每天所有预约、占用和预约状态。

关键字段：

- 预约ID
- 类型：预约 / 占用
- 预约状态
- 预约日期
- 开始时间
- 结束时间
- 客户姓名
- 手机号
- 会员ID
- 是否会员
- 服务项目
- 人员
- 房间
- 仪器
- 备注
- 创建时间
- 更新时间

## 四、会员资料表用途

用于储存会员信息，并在预约时通过手机号自动识别。

关键字段：

- 会员ID
- 客户姓名
- 手机号
- 会员等级
- 开卡日期
- 到期日期
- 套餐项目
- 剩余次数
- 余额
- 累计消费
- 最近到店日期
- 会员状态
- 备注
- 创建时间
- 更新时间

## 五、预约识别会员逻辑

1. 管理员新增预约。
2. 输入手机号。
3. 系统用手机号查找 `会员资料`。
4. 如果匹配成功：
   - 预约表单显示会员信息。
   - 自动写入会员ID。
   - 预约看板卡片显示会员等级。
   - 导出预约表时显示“是否会员 = 是”。
5. 如果匹配失败：
   - 显示非会员提示。
   - 可到 `会员系统管理` 新增会员。

## 六、接口部署步骤

1. 新建 Google Sheets。

2. 打开 Apps Script。

   路径：`扩展程序` -> `Apps Script`

3. 将 `google-apps-script.gs` 全部复制到 Apps Script。

4. 保存项目。

5. 运行一次 `setupSheets`。

   系统会自动创建：

   - `预约记录`
   - `会员资料`
   - `系统设置`

6. 授权 Apps Script。

7. 部署 Web App。

   设置建议：

   - 执行身份：我
   - 访问权限：知道链接的任何人

8. 复制 Web App URL。

9. 打开 `index.html`。

10. 点击右上角 `修改表格`。

11. 设置：

   - 数据储存方式：Google Sheets
   - Google Apps Script 接口：粘贴 Web App URL

12. 点击 `保存设置`。

13. 点击 `同步云端`。

## 七、日常使用 SOP

### 预约管理

1. 进入 `预约系统管理`。
2. 选择日期。
3. 点击空白时间格新增预约。
4. 输入客户手机号。
5. 系统自动识别是否会员。
6. 填写服务项目、人员、时长等信息。
7. 保存预约。
8. 如开启 Google Sheets 模式，系统会自动尝试同步。

### 会员管理

1. 进入 `会员系统管理`。
2. 点击 `新增会员`。
3. 填写姓名、手机号、会员等级、套餐、剩余次数、余额等信息。
4. 保存会员。
5. 后续预约时输入同一手机号，即可自动识别会员。

### 云端同步

- `读取云端`：从 Google Sheets 拉取最新数据。
- `同步云端`：将当前看板数据写入 Google Sheets。
- `导出 Excel`：导出当前预约数据。
- `导出会员 Excel`：导出会员资料。

## 八、重要注意事项

- 第一阶段适合单店、轻量使用。
- 手机和电脑都可以打开页面，但正式多人同时使用前，建议升级权限和冲突处理。
- 如果多人同时编辑，最后一次同步可能覆盖前一次同步。
- Apps Script URL 属于内部接口，不建议公开给外部客户。
- 如果浏览器阻止本地 HTML 调用接口，建议把 HTML 也部署到 Apps Script 或 HTTPS 网页。
