# 预约与会员系统 Google Sheets 储存方案说明

## 一、方案目标

第一阶段先使用 Google Sheets 作为数据储存后台，用于承接门店的预约管理与会员管理。

这样做的好处是：

- 不需要一开始搭建复杂服务器，成本低，上线快。
- 甲方可以直接打开 Google Sheets 查看、筛选、备份和导出 Excel。
- 预约系统和会员系统的数据结构可以先跑通，后续再升级为小程序或正式数据库。
- HTML 看板可以通过接口把数据写入 Google Sheets，也可以从 Google Sheets 读取最新数据。

## 二、数据表结构

建议建立一个 Google Sheets 文件，文件名为：

`门店预约与会员管理数据库`

里面包含两个主要工作表：

1. `预约记录`
2. `会员资料`

后续如果需要统计，也可以增加：

3. `服务项目设置`
4. `员工设置`
5. `操作日志`

第一阶段先做前两个表即可。

## 三、预约记录表

工作表名称：`预约记录`

建议字段如下：

| 字段 | 说明 |
|---|---|
| 预约ID | 系统自动生成，唯一标识一条预约 |
| 预约日期 | 例如 2026-06-01 |
| 开始时间 | 例如 10:00 |
| 结束时间 | 例如 12:00 |
| 客户姓名 | 客户姓名 |
| 手机号 | 用于匹配会员 |
| 是否会员 | 是 / 否 |
| 会员ID | 如果是会员，则关联会员资料表 |
| 服务项目 | 面部护理、身体护理、补水、抗衰等 |
| 美容师/顾问 | 负责人员 |
| 房间 | 可选 |
| 仪器 | 可选 |
| 预约状态 | 待确认、已确认、已到店、已完成、已取消 |
| 备注 | 特殊需求或门店备注 |
| 创建时间 | 系统自动记录 |
| 更新时间 | 系统自动记录 |

## 四、会员资料表

工作表名称：`会员资料`

建议字段如下：

| 字段 | 说明 |
|---|---|
| 会员ID | 系统自动生成，唯一标识会员 |
| 客户姓名 | 会员姓名 |
| 手机号 | 会员唯一匹配依据，建议必填 |
| 会员等级 | 普通会员、VIP、黑金卡等 |
| 开卡日期 | 会员开通日期 |
| 到期日期 | 可选 |
| 套餐项目 | 购买的服务项目或套餐 |
| 剩余次数 | 可选，用于次卡 |
| 余额 | 可选，用于储值卡 |
| 累计消费 | 可选 |
| 最近到店日期 | 可选 |
| 会员状态 | 有效、即将到期、已过期、暂停 |
| 备注 | 皮肤情况、偏好、禁忌、服务记录等 |
| 创建时间 | 系统自动记录 |
| 更新时间 | 系统自动记录 |

## 五、预约与会员的关联逻辑

预约时，系统会根据客户手机号匹配会员资料。

逻辑如下：

1. 管理员新增预约。
2. 输入客户手机号。
3. 系统查询 `会员资料` 表。
4. 如果手机号存在：
   - 显示“会员”标签。
   - 自动带出会员姓名、会员等级、剩余次数、余额、备注等信息。
   - 预约记录中写入 `是否会员 = 是` 和对应 `会员ID`。
5. 如果手机号不存在：
   - 显示“非会员”。
   - 管理员可以正常保存预约。
   - 也可以选择“新增为会员”。

这样前台在预约时就能马上知道客户是不是会员，方便安排服务和核对权益。

## 六、HTML 看板如何使用 Google Sheets

第一阶段推荐使用以下流程：

1. 甲方使用 HTML 看板进行日常预约管理。
2. 新增预约后，数据自动写入 Google Sheets 的 `预约记录` 表。
3. 新增或编辑会员后，数据自动写入 Google Sheets 的 `会员资料` 表。
4. 打开 HTML 看板时，系统从 Google Sheets 读取最新预约与会员数据。
5. 点击导出 Excel 时，可以导出当前 Google Sheets 数据。

## 七、技术连接方式

推荐使用 Google Apps Script 作为中间接口。

结构如下：

`HTML 看板 / 小程序`

连接到：

`Google Apps Script 接口`

再读写：

`Google Sheets 数据库`

这样做的原因：

- HTML 页面不需要直接暴露 Google 账号权限。
- Google Apps Script 可以控制哪些数据允许读取和写入。
- 后续小程序也可以调用同一套接口。
- 数据仍然保存在甲方容易查看的 Google Sheets 中。

## 八、甲方日常使用方式

甲方日常只需要使用两个入口：

### 1. HTML 看板

用于日常操作：

- 查看预约排班
- 新增预约
- 修改预约
- 新增占用
- 查看会员状态
- 新增或编辑会员

### 2. Google Sheets

用于数据管理和备份：

- 查看完整预约记录
- 查看完整会员资料
- 筛选某个员工、某个日期、某个会员
- 导出 Excel
- 手动备份数据

正常情况下，甲方不需要直接在 Google Sheets 里改数据；如果需要大批量整理数据，可以在 Google Sheets 中操作。

## 九、权限建议

建议 Google Sheets 权限设置为：

- 店主/管理员：可编辑
- 前台/员工：通过 HTML 看板操作，不直接编辑表格
- 外部人员：无权限

这样可以减少误删、误改数据的风险。

如果后续有多个员工同时使用，建议增加登录或员工身份字段，记录是谁创建或修改了预约。

## 十、第一阶段交付范围

第一阶段建议实现：

- Google Sheets 数据表创建
- 预约记录写入 Google Sheets
- 会员资料写入 Google Sheets
- HTML 看板读取预约记录
- HTML 看板读取会员资料
- 预约时按手机号识别会员
- 保留导出 Excel 功能

暂不建议第一阶段做复杂功能，例如：

- 在线支付
- 短信提醒
- 微信登录
- 多门店权限
- 完整财务统计

这些可以放到第二阶段。

## 十一、后续升级方向

当预约量和会员量变大后，可以升级为：

- 小程序版本
- 员工登录权限
- 会员消费记录
- 套餐扣次
- 储值余额
- 自动提醒
- 数据报表
- 正式数据库，例如 Supabase 或 Firebase

Google Sheets 第一阶段的作用，是先用最低成本把业务流程跑通，并让甲方能够真实开始使用。
